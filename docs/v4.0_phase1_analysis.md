## 波動分析：為什麼 Macro F1 在 0.52~0.54 之間劇烈震盪？

### 根本原因：Fear 的 F1 極度不穩定

```
Fear F1 歷程: 0.18 → 0.17 → 0.21 → 0.21 → 0.18 → 0.19 → 0.16 → 0.17 → 0.19 → 0.18 → 0.17 → 0.15
                                   ↑ 最高                          ↓ 最低
                                 差距: 0.06 (相對波動 37.5%)
```

對比其他三個類別的穩定度：
| 類別 | F1 範圍 | 波動幅度 | 穩定度
|:---|:---:|:---:|:---|
| Neutral | 0.75~0.78 | 0.03 | ✅ 極穩定 |
| Happy | 0.76~0.78 | 0.02 | ✅ 極穩定 |
| Surprise | 0.37~0.41 | 0.04 | ✅ 穩定 |
| **Fear** | **0.15~0.21** | **0.06** | ❌ 嚴重不穩定 |

### 為什麼 Fear 會這麼不穩定？

> [!IMPORTANT]
> **核心原因：Fear 在 Dev 只有 285 筆，佔全部 17,115 筆的 1.66%。**
> 
> 在這種極端稀少的狀況下，模型只要多猜對或猜錯 **10~15 筆** Fear 樣本，F1 就會產生巨大的跳動。這不是模型的問題，而是**統計學上的必然現象**（小樣本的高方差效應）。

具體試算：
- Dev 中 Fear 只有 285 筆
- Epoch 4 猜對 49 筆 (Recall=0.17) → F1=0.21
- Epoch 12 猜對 40 筆 (Recall=0.14) → F1=0.15
- **只差了 9 筆**，F1 就掉了 0.06，連帶 Macro F1 整體掉了 0.015

### 為什麼 Train Loss 持續下降但 Val 指標沒有跟著上？

| 觀察 | 數值 | 結論 |
|:---|:---|:---|
| Train Loss | 0.885 → 0.702 (↓ 21%) | 模型持續在學習 |
| Val Loss | 0.881 → 0.852 (↓ 3.3%) | 驗證集幾乎沒改善 |
| Train Acc | 74.9% → 78.2% (↑ 3.3%) | 訓練集越來越好 |
| Val Acc | 73.4% → 74.3% (↑ 0.9%) | 驗證集微幅改善 |

> [!NOTE]
> **結論：Overfitting 已經出現。** 66K 參數的小頭在 Epoch 1~2 就學到了最佳的線性分界面。後續 10 個 Epoch 的 Train Loss 下降，全部是在「死背」訓練集中的個別樣本，這些記憶無法轉移到驗證集。

## Phase 1 結論

### ✅ 成功的部分
1. **Macro F1 從 0.3764 → 0.5401**，提升 43.5%
2. **Neutral / Happy 達到 0.77~0.78**，已接近天花板
3. **Surprise 從 0.21 → 0.39~0.41**，翻倍成長
4. 整體 Accuracy 穩定在 **73~75%**

### ❌ 瓶頸
1. **Fear F1 卡在 0.15~0.21**，無法突破。這是因為 256 維的凍結特徵空間中，Fear 和其他情緒的邊界太模糊，光靠一個小頭劃線劃不清楚。
2. **Macro F1 的天花板 = 0.54**，完全被 Fear 拖累。若 Fear F1 能從 0.18 提升到 0.35，Macro F1 就能跳到 0.58。

### 🎯 Phase 2 的期望
解凍 `model_seq`，讓 Conv1d 重新學習如何把 1280 維 Whisper 特徵壓縮成 **更適合 4 類區分的** 256 維空間，特別改善 Fear 的特徵分離度。
