# Discriminative Learning Rates 實作完成

已為 `FusionSpeakerModel` 實作差異化學習率功能，允許針對不同模組設定不同的學習率乘數 (Multiplier)。

## 實作細節

### 1. 核心邏輯
- **新增 `lr_utils.py`**:
  - 負責遍歷模型參數，根據參數名稱前綴將其分類到不同群組。
  - 支援 DDP (`module.` 前綴) 與單機模式。
- **修改 `train.py`**:
  - 在建立 Optimizer 前，檢查 YAML 設定是否啟用 `discriminative_lr`。
  - 若啟用，使用 `build_param_groups` 分配每個群組的 `lr`。

### 2. 參數分組與預設乘數
設定檔中已預設以下分組策略：

| 群組名稱 | 包含層級 | 建議乘數 | 用途 |
|:---:|:---|:---:|:---|
| `w2v2_backbone` | Wav2Vec2 主幹 (Encoder) | **0.1x** | 預訓練特徵微調 (保守) |
| `w2v2_head` | Wav2Vec2 BN, Pooling, Projection | **0.5x** | 特徵適配層 |
| `resnet_backbone` | ResNet 主幹 (Conv Layers) | **0.2x** | 預訓練特徵微調 |
| `resnet_head` | ASP Pooling, Bottleneck | **0.5x** | 特徵適配層 |
| `fusion` | LayerNorm (Fusion 層) | **1.0x** | 新訓練層 (積極) |
| `language_head` | GRL 分類器 (MLP) | **1.0x** | 對抗訓練層 |
| `projection` | ArcFace Projection | **1.0x** | 分類層 |
| `other` | 其他未歸類參數 | **1.0x** | 預設 |

### 3. 如何使用
直接在 YAML 設定檔 (如 `conf/fusion_example.yaml`) 中調整 `discriminative_lr` 區塊：

```yaml
discriminative_lr:
  enabled: true  # 設為 false 即可關閉，恢復全域一致 LR
  lr_groups:
    w2v2_backbone: 0.1
    # ... 其他群組 ...
```
