# V4.0: 4-Class 面試情緒偵測 — 兩階段突破計畫

## Phase 1: Frozen Head Only (目前執行中 ✅)

凍結所有預訓練權重，只訓練 4 類分類頭。

```
Whisper Encoder (🔒) → model_seq Conv1d (🔒) → 256 維 → emotion_layer (🔥 66K params)
```

### Phase 1 成績 (截至 Epoch 2)
| 指標 | Epoch 1 | Epoch 2 | vs 8-class Baseline |
|:---|:---:|:---:|:---|
| **Macro F1** | **0.5239** | 0.5196 | ✅ **+39%** (Baseline: 0.3764) |
| Val Acc | 73.35% | 72.16% | — |
| Neutral F1 | 0.77 | 0.77 | ↑ from 0.69 |
| Happy F1 | 0.77 | 0.76 | ↑ from 0.54 |
| Fear F1 | 0.18 | 0.17 | ↑ from 0.06 |
| Surprise F1 | 0.37 | 0.38 | ↑ from 0.21 |

### Phase 1 修正紀錄
- ❌ 原始版本使用 `WeightedRandomSampler` + raw 逆頻率 Class Weights → Neutral F1 歸零
- ✅ 移除 `WeightedRandomSampler`，改用 `sqrt(逆頻率)` Class Weights → 四類分布正常

### Phase 1 瓶頸分析
模型在 Epoch 1 即趨於收斂，因為：
- 只有 66K 可訓練參數，學習容量有限
- 256 維特徵空間是為 8 類任務訓練的，部分維度對 4 類無用
- **Fear 的 F1 (0.18) 是拖累 Macro F1 的最大瓶頸**

## Phase 2: 解凍 model_seq (待執行 ⏳)

> [!IMPORTANT]
> **觸發條件**：Phase 1 訓練完成後，若 Macro F1 沒有顯著超過 0.53，則啟動 Phase 2。

### 核心改動
解凍 `model_seq` (3 層 Conv1d)，讓 256 維特徵空間針對 4 類任務重新優化。

```
Whisper Encoder (🔒) → model_seq Conv1d (🔥) → 256 維 → emotion_layer (🔥)
                       ↑ 解凍！~650K params        ↑ 持續訓練
```

### 為什麼這招會有效？
1. 原本的 `model_seq` 是用 8 類訓練出來的，它的 256 維空間裡可能有些維度是專門用來區分 Angry vs Disgust 的，對我們的 4 類來說是「浪費的頻寬」
2. 解凍後，Conv1d 可以重新分配這些維度，讓 Fear 和 Surprise 獲得更獨立的特徵空間
3. 總可訓練參數 ~720K，依然非常輕量，不會 Overfitting

### 超參數調整
```json
{
    "learning_rate": 0.0003,
    "weight_decay": 0.01
}
```
- **LR 降為 0.0003**：因為 `model_seq` 已經有不錯的預訓練權重，用太大的 LR 會把學好的東西打散
- 可選用 Discriminative LR：`model_seq` 用 1e-4，`emotion_layer` 用 1e-3

### 程式碼改動 (train_4class.py)
解凍 `model_seq`：
```python
# Phase 2: 解凍 model_seq
for param in model.model_seq.parameters():
    param.requires_grad = True
```

降低 LR：
```python
optimizer = torch.optim.AdamW([
    {'params': model.model_seq.parameters(), 'lr': 1e-4},   # Conv1d: 小步微調
    {'params': model.emotion_layer.parameters(), 'lr': 1e-3}, # Head: 正常學習
], weight_decay=0.01)
```

### 預期成效
- **Macro F1**: 從 0.52 → 預期 0.55~0.60
- **Fear F1**: 從 0.18 → 預期 0.25~0.35 (最大改善空間)
- **Surprise F1**: 從 0.38 → 預期 0.40~0.50

## Phase 3: 解凍 Whisper 最後 2 層 (備案 🔮)

如果 Phase 2 仍然無法突破 0.60，可以考慮解凍 Whisper Encoder 的最後 2 層 Transformer Block。

```
Whisper Encoder [..., Layer 31 (🔥), Layer 32 (🔥)] → model_seq (🔥) → emotion_layer (🔥)
```

- LR: Whisper 層用 **1e-5**（極小步伐）
- 風險：可能破壞預訓練特徵
- 回報：讓 Encoder 本身也學會「面試場景」的聲學特徵

> [!WARNING]
> 此方案風險較高，建議作為最後手段。若 Phase 2 已達 0.55+ 則不需要執行。
