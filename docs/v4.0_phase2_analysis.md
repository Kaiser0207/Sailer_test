## 分析 1：Overfitting 比 Phase 1 更快更嚴重

### Train Loss vs Val Loss 對比

```
Train Loss: 0.861 → 0.405 (↓ 53%)  ← 猛烈下降，模型瘋狂背書
Val Loss:   0.874 → 1.825 (↑ 109%) ← 驗證集 Loss 翻倍暴漲
```

| 觀察 | Phase 1 | Phase 2 | 差異 |
|:---|:---:|:---:|:---:|
| Train Loss 降幅 | -21% | **-53%** | Phase 2 下降快 2.5 倍 |
| Val Loss 變化 | -3.3% | **+109%** | Phase 2 暴漲！ |
| Overfitting 起始點 | Epoch 4 | **Epoch 3** | Phase 2 更早 |
| 最佳 Epoch | Epoch 4 | Epoch 7 | 但 Epoch 7 的 Val Loss 已到 1.01 |

> [!WARNING]
> **Phase 2 的核心矛盾**：可訓練參數增加 8 倍 (66K → 526K)，讓模型有更強的「記憶力」，但這份記憶力全部用來背訓練集，而非學到泛化的特徵。Val Loss 從 Epoch 3 起就開始單調上升，代表模型的「自信」越來越偏離真實。

## 分析 2：Fear 依然是不可逾越的牆

### Fear F1 歷程比較

```
Phase 1: 0.18 → 0.17 → 0.21 → 0.21 → 0.18 → 0.19 → 0.16 → 0.17 → 0.19 → 0.18 → 0.17 → 0.15
Phase 2: 0.20 → 0.20 → 0.20 → 0.22 → 0.19 → 0.20 → 0.21 → 0.21 → 0.21 → 0.19 → 0.19 → 0.17 → 0.16 → 0.16
```

| 指標 | Phase 1 | Phase 2 | 結論 |
|:---|:---:|:---:|:---|
| Fear F1 最高 | 0.21 | **0.22** | 僅提升 0.01 |
| Fear F1 最低 | 0.15 | 0.16 | 基本持平 |
| Fear F1 平均 | ~0.18 | ~0.20 | 略微提升 |
| Fear F1 波動幅度 | 0.06 | 0.06 | 完全一樣 |

> [!IMPORTANT]
> **結論：解凍 model_seq 對 Fear 幾乎無效。**
> 
> model_seq 負責把 Whisper 的 1280 維壓縮成 256 維。但如果 Whisper 輸出的 1280 維特徵本身就無法區分 Fear 和其他情緒，那不管怎麼重新壓縮，Fear 依然會被混淆。
>
> **瓶頸已確認在上游——Whisper Encoder 本身的特徵表達。**

## Phase 2 結論

### ✅ 成功的部分
1. **Macro F1 從 0.5401 → 0.5451**，小幅刷新紀錄
2. **Surprise Recall 明顯提升** (0.38 → 0.45+)
3. **Neutral / Happy 完全沒受損**，Discriminative LR 策略正確
4. **確認了瓶頸所在**：不在 model_seq，而在 Whisper Encoder 的特徵表達

### ❌ 瓶頸
1. **Fear F1 幾乎沒動** (0.21 → 0.22，僅 +0.01)
2. **Overfitting 比 Phase 1 更嚴重**：Val Loss 在 14 個 Epoch 內翻了 2 倍
3. **總體 Macro F1 提升幅度微小** (+0.005)，不足以視為突破

### 🔑 關鍵洞察

> Fear 的問題不是「下游分類器不夠強」，而是「上游 Whisper 給出的特徵本身就沒有把 Fear 和 Neutral/Happy 分開」。
>
> 解法：必須讓 Whisper Encoder 的最後一層開始「學會聽見恐懼」，同時用 Focal Loss 把訓練的注意力強制集中在 Fear 上。

### 🎯 Phase 3 方向
1. **Focal Loss**：取代 CrossEntropyLoss，抑制簡單樣本的梯度，聚焦 Fear
2. **解凍 Whisper Encoder 最後 1 層**：讓 1280 維特徵空間本身重新學習，從根源解決 Fear 的可分離性問題
